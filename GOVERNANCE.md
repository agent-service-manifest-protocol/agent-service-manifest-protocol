# Governance

## Current Stage: Incubation

ASMP (Agent Service Manifest Protocol) is currently in incubation — single maintainer, proving the spec against real infrastructure.

### Maintainer

- Daniel Shanklin (@dshanklin-bv)

### Roadmap to Neutral Governance

**Stage 1 — Incubation (current)**
- Single maintainer
- Spec development and iteration
- Reference implementation
- Prove on 35+ real services (Reeves ecosystem)

**Stage 2 — Community Adoption**
- 3-5 external projects using ASMP
- Community contributions accepted
- SDKs in Python and TypeScript
- Steering committee formed (3-5 members)

**Stage 3 — Foundation Governance**
- Submit to Linux Foundation / Agentic AI Foundation
- Establish as "Agent Service Manifest Protocol, a Series of LF Projects, LLC"
- Neutral governance, shared infrastructure
- Trademark protection via LF Projects policies

### Decision Process

During incubation, the maintainer makes all decisions. As the project moves to Stage 2, decisions will be made by consensus of the steering committee. At Stage 3, governance follows LF Projects policies.

### Complementary Protocols

ASMP is designed to compose with, not compete against:
- **MCP** (Model Context Protocol) — tool discovery and invocation
- **A2A** (Agent-to-Agent Protocol) — inter-agent communication
- **ACP** (Agent Communication Protocol) — agent messaging

ASMP operates at the host layer — where services run, how they're registered, and how agents extend them.
